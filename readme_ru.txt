	Для запуска сборки проекта необходимо перейти в каталог build и запустить скрипты.
Перед запуском скриптов убедитесь, что установлены JDK версии не ниже 5.0 (рекомендуется
использовать данную версию), ANT версии не ниже 1.6.5 (на файлы запуска ant должны быть
проложены маршруты). Необходимо создать переменную окружения JAVA_HOME содержащую
путь установки JDK.
	Cкрипт build (sh для Linux, bat для Windows) используется для компиляции и сборки.
После выполнения скрипта в каталогах build/output, workbench/plugins будут находиться
собранные файлы. Содержимое каталогов build/output/ejb, build/output/lib, build/output/ui,
build/output/service, build/output/web предназначено для развертывания на сервере
приложения. Содержимое каталога workbench/plugins предназначено для развертывания
в среде разработки.
	Скрипт build_docs используется для сборки поставляемой документации. В каталоге
docs/output будет находиться собранная документация. Если в файле build.properties
установить свойство build.docs.pdf в любое значение, то дополнительно будут созданы
файлы в формате pdf (по умолчанию данная опция отключена).
	Скрипт build_installer производит сборку инсталлятора системы. Однако для данного
действия необходимо иметь в наличии дистрибутивы программного обеспечения сторонних
производителей (JBoss, Eclipse, BIRT runtime), данные дистрибутивы рекомендуется загрузить
с сайта www.m-g.ru. В файле свойств build.properties необходимо указать пути к данным
дистрибутивам. В каталоге release\product\output будет находиться инсталлятор.
