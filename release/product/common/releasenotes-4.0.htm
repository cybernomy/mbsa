Добавлена поддержка Java 6.0.

Реализован монитор сессий пользователей, монитор содержит подробную информацию о подключенных пользователях и позволяет администратору системы производить рассылку сообщений пользователям. В текущей версии не поддерживается атрибут "Размер сессии" и его значение всегда равно 0.

Реализован планировщик заданий, предназначен для запуска различных заданий по расписанию, подробная информация находится в контекстной помощи и документации администратора системы.

Параметры отчетов сортируются в соответствии с порядком установленном в шаблоне отчета.

Параметры запуска отчетов сохраняются в профиле пользователя.

Реализован интерактивный визуализатор отчетов, позволяет переформировывать отчеты, экспортировать в различные форматы, выполнять печать.

Формат вывода числовых значений соответствует текущим языковым настройкам пользователя.

Реализован динамический разделитель в формах поддержки, позволяет сворачивать области форм связанных с разделителем.

Реализован контроль изменений в формах поддержки, если были изменения и пользователь пытается закрыть форму путем нажатия Ctrl+F4 или кнопки X меню формы, то будет выдан запрос с вариантами дальнейших действий.

Реализована назначение прав пользователям на все компоненты модуля, предназначен для упрощения процедура настройки прав пользователя.

Реализована поддержка L&F Substance, альтернативный набор расширений пользовательского интерфейса, требует наличия на стороне клиента JRE 6.0 и выше. Для использования данного L&F необходимо использовать следующие адреса: http://serverhost:8080/mbsaclient/merpclientsb.jnlp или http://serverhost:8080/mbsaclient/merpclientsb.jsp

Реализована поддержка "внутренних" пользователей системы, без права интерактивного взаимодействия, примером такого пользователя может служить scheduler используемый для выполнения заданий планировщика.

Реализована поддержка выбора доступного языка при подключении к системе. Подробная информация о настройке доступных языков находится в документации администратора системы.

В поставку включено приложение JBoss Administration Console предназначенное для мониторинга и управления сервером приложения.

В качестве платформы для модуля BPM используется JBPM 3.2.2. В студию разработки включен jBPM designer 3.1.0sp1

Новые функциональные возможности

Журналы аудита позволяют администратору системы отслеживать различные события: вход и выход пользователей; создание, изменение и удаление данных, запуск документооборота и т.п.

Созданы новые разделы системы: Балансовые единицы (модуль Core), Виды договора и Категории договора (модуль Contract), Тип склада (модуль Warehouse).

Перемещение записей по папкам и изменение иерархии папок реализовано с помощью функциональности Drag&Drop. Зависит от прав пользователя на данные операции, при отсутствии прав действия не выполняются. На перенос папок действуют те же правила, что и в версии 2.х, но визуально курсор операции не меняется. Если операция невозможна, действие не происходит.

Реализовано копирование записей. Для некоторых объектов работает частичное или полное (с деталями) копирование. Для простых объектов, не имеющих детали, обе функции идентичны.

Реализовано применение скидок и наценок в исходящих товарных документах.

Константы. Реализовано, как в предыдущих версиях.

Справочник "Физические лица": При добавлении новой записи физического лица все детали (история) недоступны, пока не создан главный объект (мастер). При добавлении новой записи автоматически формируется первая строка Истории. Поля "Фамилия", "Имя", "Отчество", "Действует с" и "ИНН" становятся недоступными для редактирования. Изменить их можно, добавив новую запись истории изменения сведений с большей датой. При удалении актуальной записи истории изменений поля физического лица принимают значения второй по актуальности записи истории изменений. Последнюю запись истории изменений удалить нельзя.

Справочник "Праздничные дни", добавлена функция "Скопировать на следующий год".

Модуль "Зарплата". Реализованы основные функции: расчёт зарплаты, создание платёжной ведомости, создание свода начислений/удержаний по аналитике

Модуль "Планирование производства":

  • Генерация Основного производственного плана (ОПП), Перенос ОПП, Расчёт кодов нижнего уровня.
  • В Логистическом маршруте поля "Тип источника товара" и "Тип приёмника товара" стали обязательными.

Модуль "Производство":

  • в разделе Заказ-наряды на производство (ЗНП) реализованы функции "Запустить", "Остановить", "Завершить".
  • Рабочие центры: в браузере удалено несколько полей, не использующихся в текущей версии.

Справочники учёта персонала:

  • Реализация работы с адресами по КЛАДР ("Основные сведения о сотрудниках")
  • Изменена проверка дат при добавлении, изменении расчётного периода.

Типы документов: удалено поле "Шаблон документа по умолчанию".

Управление бизнес-процессами: обработаны сообщения об ошибках и установлена проверка обязательных полей при добавлении записей.

Розничные документы и дисконтные карты. Реализовано, как было в предыдущих версиях: при выборе партнёра автоматически указывается его дисконтная карта, либо появляется список связанных с ним дисконтных карт.

Дисконтные карты: при добавлении истории скидок поле "Оператор" автоматически заполняется именем текущего пользователя.

Карточки складского учёта: просмотр складских партий.

Управление качеством: сделана проверка обязательных полей при добавлении записей.

Платёжный календарь:

  • в разделе "Управление платежами" добавлены Условия отбора для списка Средств платежа;
  • в списке обязательств добавлены поля "Исполнено на сумму" и "Остаток".

Графики исполнения обязательств: реализована возможность просмотра документа, для которого создан ГИО (пункт в контекстном меню и кнопка в форме поддержки).

CRM. База знаний, Решения: добавлены Условия отбора.

Финансовый учёт:

  • Финансовые операции:
    • Курс валюты устанавливается стандартной функцией, как и в других разделах системы.
    • Сумма операции отображается в браузере.
    • Признак "План" устанавливается при создании операции и в дальнейшем не может быть изменён, а также сохраняется при копировании.
  • В ведомостях "Остатки и обороты по счетам финансового учёта" и "Остатки и обороты по признакам финансового учёта" добавлены поля для подсчёта разницы между планируемыми и фактическими оборотами.

Документы и контракты: в Услових отбора исправлен механизм фильтра по номеру.

Бухгалтерский учёт:

  • Книга покупок, книга продаж: реализовано добавление записей.
  • Периоды бухгалтерского учёта - установлена проверка обязательных полей при добавлении записи.

Контекстная помощь по клавише F1: подключена в разделе "Типы документов" и карте настройки документооборота. Создана помощь для новых объектов системы (Журналы аудита, Балансовая единица, Тип склада, Вид договора и Категория договора).
